# Phase 4.2: Formulário de Avaliação — Research

**Researched:** 2026-06-07
**Domain:** Formulário inline com CRUD — Next.js App Router + SQLite + shadcn/ui
**Confidence:** HIGH

## Summary

Esta fase implementa a página de formulário de avaliação individual com 6 categorias (escala 1-10), média automática via SQLite GENERATED column, comentário opcional, listagem com filtros e detalhes apenas-leitura em modal.

**Padrões existentes da Fase 4.1 são a referência:** API routes com try/catch + prepared statements, server component page + client component, shadcn Dialog para modais, nomes de arquivos kebab-case.

**Primary recommendation:** Seguir o pattern de `funcionarios-table.tsx` para a listagem com filtros, mas com `mode` state para alternar entre listagem e formulário inline (em vez de modal). Usar o novo shadcn Combobox (v4 radix-nova) para o dropdown com busca de funcionário. Usar `<input type="date">` nativo (padrão existente em `period-filter.tsx`). Média calculada automaticamente no SQLite via `GENERATED ALWAYS AS ... STORED`.

**Sidebar:** Item "Avaliação" já existe em `sidebar-items.ts` (linha 56-59) com `url: "/dashboard/avaliacao"` e **SEM** `comingSoon` — nenhuma modificação necessária na sidebar.

### Pattern 2: Client Component com Mode State (List vs Form)

**What:** Componente que gerencia o estado de navegação entre listagem e formulário inline.

**When to use:** Quando D4.2-07 exige formulário na mesma página (não modal, não rota separada).

```tsx
"use client";

import { useState } from "react";
import { AvaliacoesTable } from "./avaliacoes-table";
import { AvaliacaoForm } from "./avaliacao-form";

type PageMode = "list" | "form";

export function AvaliacoesPageClient() {
  const [mode, setMode] = useState<PageMode>("list");
  const [refreshKey, setRefreshKey] = useState(0);

  function handleNovaAvaliacao() {
    setMode("form");
  }

  function handleFormCancel() {
    setMode("list");
  }

  function handleFormSuccess() {
    setMode("list");
    setRefreshKey((k) => k + 1); // trigger refetch
    // toast via sonner
  }

  if (mode === "form") {
    return (
      <AvaliacaoForm
        onCancel={handleFormCancel}
        onSuccess={handleFormSuccess}
      />
    );
  }

  return (
    <>
      <AvaliacoesTable
        refreshKey={refreshKey}
        onNovaAvaliacao={handleNovaAvaliacao}
      />
    </>
  );
}
```

### Pattern 3: API Route — GET list com filtros + POST create (sem media)

**What:** API route que aceita query params para filtragem e INSERT sem incluir a coluna `media` (GENERATED column).

**Source:** `src/app/api/funcionarios/route.ts` [VERIFIED: codebase]

```typescript
// src/app/api/avaliacoes/route.ts
import { type NextRequest, NextResponse } from "next/server";
import { getDb } from "@/lib/db";

export async function GET(request: NextRequest) {
  try {
    const { searchParams } = new URL(request.url);
    const funcionarioId = searchParams.get("funcionario_id");
    const from = searchParams.get("from");
    const to = searchParams.get("to");

    const db = getDb();
    const conditions: string[] = [];
    const queryParams: (string | number)[] = [];

    if (funcionarioId) {
      conditions.push("a.funcionario_id = ?");
      queryParams.push(Number(funcionarioId));
    }
    if (from) {
      conditions.push("a.data_avaliacao >= ?");
      queryParams.push(from);
    }
    if (to) {
      conditions.push("a.data_avaliacao <= ?");
      queryParams.push(to);
    }

    const where = conditions.length > 0 ? `WHERE ${conditions.join(" AND ")}` : "";

    const rows = db
      .prepare(
        `SELECT a.id, a.data_avaliacao, a.media, a.comentario,
                a.funcionario_id, f.nome as funcionario_nome
         FROM avaliacoes a
         JOIN funcionarios f ON f.id = a.funcionario_id
         ${where}
         ORDER BY a.data_avaliacao DESC`,
      )
      .all(...queryParams);

    return NextResponse.json(rows);
  } catch (error) {
    const message = error instanceof Error ? error.message : "Erro interno";
    return NextResponse.json({ error: message }, { status: 500 });
  }
}

export async function POST(request: NextRequest) {
  try {
    const body = await request.json();
    const { funcionario_id, data_avaliacao, pontualidade, qualidade,
            produtividade, trabalho_equipa, iniciativa, comunicacao, comentario } = body;

    // Validações
    if (!funcionario_id || !data_avaliacao) {
      return NextResponse.json(
        { error: "Funcionário e data são obrigatórios" },
        { status: 400 },
      );
    }

    const categorias = [
      ["pontualidade", pontualidade],
      ["qualidade", qualidade],
      ["produtividade", produtividade],
      ["trabalho_equipa", trabalho_equipa],
      ["iniciativa", iniciativa],
      ["comunicacao", comunicacao],
    ] as const;

    for (const [nome, valor] of categorias) {
      if (valor === undefined || valor === null || typeof valor !== "number") {
        return NextResponse.json(
          { error: `Campo ${nome} é obrigatório (1-10)` },
          { status: 400 },
        );
      }
      if (valor < 1 || valor > 10 || !Number.isInteger(valor)) {
        return NextResponse.json(
          { error: `${nome} deve ser um número inteiro entre 1 e 10` },
          { status: 400 },
        );
      }
    }

    const db = getDb();

    // INSERT sem media — GENERATED column calcula automaticamente
    const result = db
      .prepare(
        `INSERT INTO avaliacoes
         (funcionario_id, data_avaliacao, pontualidade, qualidade,
          produtividade, trabalho_equipa, iniciativa, comunicacao, comentario)
         VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)`,
      )
      .run(
        funcionario_id, data_avaliacao,
        pontualidade, qualidade, produtividade,
        trabalho_equipa, iniciativa, comunicacao,
        comentario ?? null,
      );

    // Buscar o registro inserido (incluindo media calculada)
    const created = db
      .prepare("SELECT * FROM avaliacoes WHERE id = ?")
      .get(result.lastInsertRowid);

    return NextResponse.json(created, { status: 201 });
  } catch (error) {
    const message = error instanceof Error ? error.message : "Erro interno";
    return NextResponse.json({ error: message }, { status: 500 });
  }
}
```

### Pattern 5: Migração SQLite — Tabela avaliacoes com GENERATED column

**What:** Schema completo da tabela `avaliacoes` conforme D-30, com CHECK constraints, FOREIGN KEY e GENERATED column para média.

```typescript
// Em src/lib/db.ts, dentro de runMigrations()
// Schema conforme D-30 [VERIFIED: CONTEXT.md]

database.exec(`
  CREATE TABLE IF NOT EXISTS avaliacoes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    funcionario_id INTEGER NOT NULL,
    data_avaliacao TEXT NOT NULL,
    pontualidade INTEGER NOT NULL CHECK(pontualidade >= 1 AND pontualidade <= 10),
    qualidade INTEGER NOT NULL CHECK(qualidade >= 1 AND qualidade <= 10),
    produtividade INTEGER NOT NULL CHECK(produtividade >= 1 AND produtividade <= 10),
    trabalho_equipa INTEGER NOT NULL CHECK(trabalho_equipa >= 1 AND trabalho_equipa <= 10),
    iniciativa INTEGER NOT NULL CHECK(iniciativa >= 1 AND iniciativa <= 10),
    comunicacao INTEGER NOT NULL CHECK(comunicacao >= 1 AND comunicacao <= 10),
    media REAL GENERATED ALWAYS AS (
      (pontualidade + qualidade + produtividade + trabalho_equipa + iniciativa + comunicacao) / 6.0
    ) STORED,
    comentario TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (funcionario_id) REFERENCES funcionarios(id)
  )
`);
```

### Anti-Patterns to Avoid

- **Incluir `media` no INSERT:** A coluna `media` é `GENERATED ALWAYS AS STORED` — NÃO incluir no INSERT. O SQLite rejeita INSERT com valor explícito para generated column.
- **Fazer refetch completo ao navegar entre list/form:** Usar `refreshKey` state para trigger de refetch, não recarregar página.
- **Validar notas só no client:** As CHECK constraints no SQLite são a última linha de defesa — validar também no backend da API.
- **Criar sidebar item novo:** O item "Avaliação" já existe. Não duplicar.
- **Usar react-hook-form para 6 inputs:** O formulário tem campos suficientes para justificar react-hook-form + Zod (diferente do formulário de 1 campo da 4.1). Ver seção Don't Hand-Roll.

## Common Pitfalls

### Pitfall 1: Incluir `media` no INSERT em tabela com GENERATED column

**What goes wrong:** `INSERT INTO avaliacoes (...) VALUES (..., media, ...)` — SQLite rejeita com erro "cannot INSERT into generated column 'media'".

**Why it happens:** `GENERATED ALWAYS AS ... STORED` columns não podem ter valores inseridos ou atualizados manualmente.

**How to avoid:** Não listar `media` na cláusula INSERT. Exemplo correto:
```typescript
db.prepare(
  `INSERT INTO avaliacoes
   (funcionario_id, data_avaliacao, pontualidade, qualidade,
    produtividade, trabalho_equipa, iniciativa, comunicacao, comentario)
   VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)`
).run(...);
```
Depois do INSERT, fazer um SELECT para obter o valor da media calculada.

**Warning signs:** Erro "cannot INSERT into generated column" ou "table avaliacoes has no column named media" no runtime.

### Pitfall 2: Notas Fora do Range 1-10

**What goes wrong:** Usuário insere valor 0, 11, ou texto no campo de nota — a CHECK constraint no SQLite rejeita com erro.

**Why it happens:** SQLite CHECK constraints são avaliadas no INSERT e rejeitam valores fora do range.

**How to avoid:** Tripla validação:
1. HTML: `<input type="number" min={1} max={10} step={1} />`
2. Client-side: validar antes de enviar — `if (nota < 1 || nota > 10)`
3. Backend: validar na API route antes do INSERT

**Warning signs:** Erro "CHECK constraint failed: avaliacoes" no servidor.

### Pitfall 3: Floating Promises no useEffect + fetch

**What goes wrong:** `fetch(...)` dentro de `useEffect` sem tratamento dispara erro de lint `noFloatingPromises`.

**Why it happens:** Regra `noFloatingPromises: "error"` no Biome exige que toda Promise seja awaitada ou explicitamente ignorada com `void`.

**How to avoid:** Usar `void fetch()` para fire-and-forget, ou criar função async separada:
```typescript
useEffect(() => {
  void fetchAvaliacoes();
}, [fetchAvaliacoes]);
```

### Pitfall 4: `useFilenamingConvention: "error"` — Biome

**What goes wrong:** Arquivos com nomes em camelCase como `avaliacaoForm.tsx` quebram o lint.

**How to avoid:** Usar kebab-case para **todos** os arquivos:
- ✅ `avaliacoes-table.tsx`
- ✅ `avaliacao-form.tsx`
- ✅ `avaliacao-detail-modal.tsx`
- ✅ `avaliacoes-page-client.tsx`

### Pitfall 5: Combobox com funcionários inativos no filtro de listagem

**What goes wrong:** Ao filtrar avaliações por funcionário, o dropdown mostra todos os funcionários, incluindo inativos.

**Why it happens:** O filtro da listagem deve mostrar funcionários que TÊM avaliações, mas o dropdown de filtro poderia incluir inativos.

**How to avoid:** Carregar funcionários para o filtro da listagem com `GET /api/funcionarios?ativos=true` (apenas ativos, consistente com D4.2-08). Para o dropdown do formulário, também usar `?ativos=true`.

### Pitfall 6: Média Exibida sem Arredondamento

**What goes wrong:** `media` como `REAL` no SQLite pode ter muitas casas decimais (ex: 7.333333333333333).

**How to avoid:** Arredondar no client com `media.toFixed(2)` ao exibir na tabela e detalhes.

### Pitfall 7: Ordenação da Tabela

**What goes wrong:** Avaliações aparecem em ordem aleatória ou ascendente.

**How to avoid:** Query SQL com `ORDER BY a.data_avaliacao DESC` (D4.2-03) e `ORDER BY a.id DESC` como desempate.

### Pitfall 8: Estado do Formulário não Reseta ao Alternar List/Form

**What goes wrong:** Ao abrir o formulário, cancelar, e abrir novamente, os campos ainda têm valores anteriores.

**How to avoid:** Em `avaliacao-form.tsx`, resetar todo o estado no mount (ou usar `key` baseado no timestamp de abertura). No `avaliacoes-page-client.tsx`, usar `key={Date.now()}` no componente de formulário para forçar remontagem.

### Pitfall 9: Input type="number" com Decimais

**What goes wrong:** `<input type="number">` permite valores como 7.5, mas a escala é 1-10 com inteiros.

**How to avoid:** Usar `step={1}` no input HTML para restringir a inteiros.

### API Route — GET /api/avaliacoes/[id] (detalhes)

```typescript
// src/app/api/avaliacoes/[id]/route.ts
// Pattern: src/app/api/cil/[cil]/route.ts [VERIFIED: codebase]

import { type NextRequest, NextResponse } from "next/server";
import { getDb } from "@/lib/db";

export async function GET(
  request: NextRequest,
  { params }: { params: Promise<{ id: string }> },
) {
  try {
    const { id } = await params;
    const avaliacaoId = Number(id);
    if (Number.isNaN(avaliacaoId)) {
      return NextResponse.json({ error: "ID inválido" }, { status: 400 });
    }

    const db = getDb();
    const row = db
      .prepare(
        `SELECT a.*, f.nome as funcionario_nome
         FROM avaliacoes a
         JOIN funcionarios f ON f.id = a.funcionario_id
         WHERE a.id = ?`,
      )
      .get(avaliacaoId);

    if (!row) {
      return NextResponse.json({ error: "Avaliação não encontrada" }, { status: 404 });
    }

    return NextResponse.json(row);
  } catch (error) {
    const message = error instanceof Error ? error.message : "Erro interno";
    return NextResponse.json({ error: message }, { status: 500 });
  }
}
```

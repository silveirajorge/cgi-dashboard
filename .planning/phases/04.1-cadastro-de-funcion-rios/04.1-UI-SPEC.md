## Layout & Interaction Contract

### 1. Página `/dashboard/funcionarios` (Server Component)

**Estrutura visual (top-down):**
```
div.flex.flex-col.gap-6
├── div.flex.items-center.justify-between
│   └── h1 "Funcionários" (font-semibold text-2xl)
└── <FuncionariosTable /> (Client Component)
```

### 2. Client Component — `funcionarios-table.tsx`

**Estados visuais:**

| Estado | Trigger | Renderização |
|--------|---------|--------------|
| **Loading** | `loading === true` (fetch inicial) | `<Loader2 />` centralizado com `animate-spin` dentro de `py-12` |
| **Error** | `error !== null` | `<div>` com borda `border-destructive/50`, fundo `bg-destructive/5`, texto `text-destructive text-sm` — mensagem de erro |
| **Empty** | `filtered.length === 0 && !loading` | `<TableCell colSpan={3}>` com "Nenhum funcionário encontrado" centralizado, cor `text-muted-foreground` |
| **Data** | `filtered.length > 0` | Tabela com linhas de funcionários |

**Barra de busca e filtro:**
```
div.mb-4.flex.items-center.gap-4
├── div.relative.flex-1
│   ├── <Search /> absolute left-2.5 top-1/2 -translate-y-1/2 text-muted-foreground
│   ├── <Input placeholder="Buscar por nome..." className="pl-8" />
│   └── button × (limpar, absolute right-2.5, visível só quando search !== "")
└── label.flex.items-center.gap-2
    └── input[type="checkbox"] + text "Só ativos"
```

**Comportamento de busca:**
- Filtragem **client-side** do array já carregado
- Case-insensitive: `f.nome.toLowerCase().includes(search.toLowerCase())`
- Sensível a tempo real: atualiza a cada keystroke
- Botão × aparece para limpar o campo rapidamente

**Comportamento do filtro "Só ativos":**
- Checkbox que filtra apenas `f.ativo === 1`
- Combinável com a busca por nome (AND)
- Atualização instantânea (client-side)

**Tabela:**

| Coluna | Largura | Alinhamento | Conteúdo |
|--------|---------|-------------|----------|
| Nome | Flex-1 (maioria) | Esquerda | `font-medium`, texto do nome |
| Status | Auto | Esquerda | `<Badge variant={f.ativo ? "default" : "secondary"}>` — "Ativo" / "Inativo" |
| Ações | Auto | Direita | `<Button variant="ghost" size="icon-sm">` com `<Pencil size=4 />` + `<Button variant="outline" size="xs">` "Ativar" / "Desativar" |

### 3. Modal — `funcionario-modal.tsx`

**Reutilizável** para add e edit, diferenciado pela prop `funcionario`:
- `funcionario === undefined` → modo "Adicionar"
- `funcionario !== undefined` → modo "Editar" (pré-preenchido)

**Estrutura visual:**
```
<Dialog>
  <DialogContent>
    <DialogHeader>
      <DialogTitle> "Adicionar Funcionário" | "Editar Funcionário" </DialogTitle>
      <DialogDescription> "Insira o nome do novo funcionário" | "Altere o nome do funcionário" </DialogDescription>
    </DialogHeader>
    div.flex.flex-col.gap-4.py-2
    ├── div.flex.flex-col.gap-1
    │   ├── label.text-muted-foreground.text-sm "Nome" htmlFor="nome-input"
    │   └── <Input id="nome-input" placeholder="Nome do funcionário" 
    │              value={nome} onChange={...} 
    │              onKeyDown={Enter → handleSave} />
    ├── p.text-destructive.text-xs (mensagem de erro, condicional)
    └── div.flex.justify-end.gap-2
        ├── <Button variant="outline" onClick={onClose}> Cancelar </Button>
        └── <Button onClick={handleSave} disabled={saving || !nome.trim()}> 
              {saving ? "Guardando..." : "Guardar"} 
            </Button>
  </DialogContent>
</Dialog>
```

**Comportamentos do modal:**

| Comportamento | Detalhe |
|--------------|---------|
| **Abertura** | Chamado via `setModalOpen(true)` — estado `editingFuncionario` define modo |
| **Fechamento** | Click outside (padrão Dialog), Escape, ou botão Cancelar — sem confirmação de perda de dados (aceitável para 1 campo) |
| **Reset de estado** | `useEffect` em `open` reseta `nome` e `error` |
| **Validação** | Cliente-side: nome.trim() obrigatório antes de submit. Backend: retorna erro 400 (nome vazio) ou 409 (duplicado) |
| **KeyDown Enter** | No input de nome, Enter dispara `handleSave` |
| **Saving state** | Botão "Guardando..." + disabled durante o fetch |
| **Erro inline** | Mensagem de erro exibida em `text-destructive text-xs` abaixo do input, vinda do backend |
| **Pós-sucesso** | Modal fecha, `fetchFuncionarios()` refaz GET para atualizar lista |

### 4. Toggle Ativo/Inativo na Tabela

**Mecanismo:** Botão "Ativar" / "Desativar" na coluna Ações (não Switch — Switch não está no projeto).

**Comportamento:**
| Ação | Estado Atual | Botão | Resultado |
|------|-------------|-------|-----------|
| Clicar "Desativar" | Ativo | `variant="outline" size="xs"` | `PUT { ativo: false }` — soft delete |
| Clicar "Ativar" | Inativo | `variant="outline" size="xs"` | `PUT { ativo: true }` — reativar |

**Otimistic update:**
1. Atualiza estado local imediatamente (`setFuncionarios(prev => prev.map(...))`)
2. Dispara `PUT /api/funcionarios/[id]` com `{ ativo: boolean }`
3. Se erro no fetch → reverte estado local ao original
4. **Sem refetch completo** — evita flash na UI

| Botão Guardar | `Guardar` / `Guardando...` | Primary button no modal (ativo/inativo conforme estado) |
| Botão Cancelar | `Cancelar` | Outline button no modal |

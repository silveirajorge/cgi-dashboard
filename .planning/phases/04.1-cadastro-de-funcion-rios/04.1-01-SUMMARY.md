## Decisions Made
- Partial update (PUT) accepts nome OR ativo OR both — essential for toggle without always sending nome
- Duplicate check uses SQLite LOWER() for case-insensitive, excluding own ID on edit
- Toggle uses optimistic update without full refetch to avoid UI flash (per Anti-Pattern 1 in RESEARCH.md)
- Reusable modal for add/edit with state reset via useEffect([open, funcionario]) (per RESEARCH Pitfall 2)
- Simple useState instead of react-hook-form + Zod — 1 field only, overkill
- Soft delete via ativo=0 (per D4.1-16 and D-15) — no physical row removal

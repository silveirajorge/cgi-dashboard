# Phase 4.1: Cadastro de Funcionários — Context

**Gathered:** 2026-06-07
**Status:** Ready for planning

<domain>
## Phase Boundary

Página de gestão de funcionários com tabela, cadastro, edição e soft delete. Permite listar, adicionar, editar nome e ativar/desativar funcionários. Dados armazenados em SQLite na tabela `funcionarios`.

Decisões da Fase 4 (contexto geral) aplicam-se — ver D-13 a D-18 em `04-CONTEXT.md`.

</domain>

<decisions>
## Implementation Decisions

### Estrutura da Página
- **D4.1-01:** **Página dedicada** em `/dashboard/funcionarios`
- **D4.1-02:** **Item na sidebar** no grupo "Avaliação e Desempenho" — título "Funcionários", ícone `Users`
- **D4.1-03:** **Padrão server component + client component** — `page.tsx` server + `funcionarios-table.tsx` client em `_components/`
- **D4.1-04:** **Tabela completa** com colunas: Nome, Status (Ativo/Inativo), Ações

### Layout da Listagem
- **D4.1-05:** **Busca por nome** — campo de texto que filtra a tabela em tempo real
- **D4.1-06:** **Filtro "Só ativos"** — toggle/checkbox para mostrar apenas funcionários ativos
- **D4.1-07:** Segue o padrão visual do dashboard (Card, Table do shadcn/ui)

### Adicionar / Editar
- **D4.1-08:** **Modal (Dialog)** para adicionar novo funcionário — campo nome + botão Guardar
- **D4.1-09:** **Modal (Dialog)** para editar nome — pré-preenchido, igual ao de adicionar
- **D4.1-10:** **Toggle ativo/inativo direto na tabela** — switch ou badge clicável que altera o estado

### Validação
- **D4.1-11:** Nome é **obrigatório** (sem tamanho mínimo/máximo definido)
- **D4.1-12:** **Nomes duplicados bloqueados** — validação no backend (case-insensitive) que retorna erro se o nome já existir

### API
- **D4.1-13:** `GET /api/funcionarios` — lista todos, aceita `?ativos=true`
- **D4.1-14:** `POST /api/funcionarios` — body `{ nome }`, retorna erro 409 se duplicado
- **D4.1-15:** `PUT /api/funcionarios/[id]` — body `{ nome }` ou `{ ativo: boolean }`
- **D4.1-16:** `DELETE /api/funcionarios/[id]` — soft delete (set ativo=false)

### Banco de Dados
- **D4.1-17:** Tabela `funcionarios` (ver schema em D-17 do contexto geral da Fase 4)

### the Agent's Discretion
- Ícone específico para o item da sidebar (`Users` da lucide-react)
- Layout exato da tabela (largura das colunas)
- Texto dos placeholders e botões
- Cor do badge Ativo/Inativo

</decisions>

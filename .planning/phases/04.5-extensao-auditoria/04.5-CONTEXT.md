# Phase 4.5: Extensão da Auditoria — Context

**Gathered:** 2026-06-07
**Status:** Ready for planning

<domain>
## Phase Boundary

Extensão do formulário de avaliação/auditoria com novos campos: atrasos, faltas, uso de ferramenta, erro crítico (booleanos), produtividade (%, pode exceder 100), nota de auditoria (0-100) e tipo de auditoria (supervisor/auditor em semanas intercaladas). Atualização do dashboard para incluir novos KPIs.

</domain>

<decisions>
## Implementation Decisions

### Novos Campos na Tabela `avaliacoes`
- **D4.5-01:** Migration SQLite: adicionar colunas à tabela `avaliacoes`:
  ```sql
  ALTER TABLE avaliacoes ADD COLUMN atraso INTEGER NOT NULL DEFAULT 0;
  ALTER TABLE avaliacoes ADD COLUMN falta INTEGER NOT NULL DEFAULT 0;
  ALTER TABLE avaliacoes ADD COLUMN uso_ferramenta INTEGER NOT NULL DEFAULT 0;
  ALTER TABLE avaliacoes ADD COLUMN erro_critico INTEGER NOT NULL DEFAULT 0;
  ALTER TABLE avaliacoes ADD COLUMN produtividade REAL;
  ALTER TABLE avaliacoes ADD COLUMN nota_auditoria REAL;
  ALTER TABLE avaliacoes ADD COLUMN tipo_auditoria TEXT CHECK(tipo_auditoria IN ('supervisor', 'auditor'));
  ```
  Nota: `avaliacoes` foi criada com `CREATE TABLE IF NOT EXISTS` — usar ALTER TABLE com IF NOT EXISTS ou capturar erro se a coluna já existir.

### Formulário
- **D4.5-02:** Adicionar ao formulário `avaliacao-form.tsx`:
  - 4 checkboxes/toggles: Atraso, Falta, Uso de Ferramenta, Erro Crítico
  - Campo numérico: Produtividade (%), sem limite superior
  - Campo numérico: Nota da Auditoria (0-100)
  - Radio/Select: Tipo de Auditoria (Supervisor / Auditor)
- **D4.5-03:** A nota_auditoria e tipo_auditoria são campos obrigatórios

### API
- **D4.5-04:** `POST /api/avaliacoes` — aceitar novos campos no body
- **D4.5-05:** `GET /api/avaliacoes` — retornar novos campos no SELECT
- **D4.5-06:** `GET /api/avaliacoes/stats` — incluir novos KPIs:
  - `media_nota_auditoria` — média das notas de auditoria no período
  - `total_atrasos` — soma de atrasos
  - `total_faltas` — soma de faltas
  - `total_erros_criticos` — soma de erros críticos
  - `media_produtividade` — média da produtividade
  - `comparativo_auditoria` — média por tipo (`supervisor` vs `auditor`)

### Dashboard
- **D4.5-07:** Adicionar novos cards de KPI no dashboard de avaliação (Fase 4.3):
  - Média Nota Auditoria
  - Total Atrasos / Faltas
  - Erros Críticos
  - Média Produtividade
- **D4.5-08:** Gráfico comparativo: nota auditoria supervisor vs auditor (barras lado a lado)

</decisions>

## Interaction Contracts

### 1. Data Loading

| Trigger | Behavior | Visual Feedback |
|---------|----------|-----------------|
| Page mount | Fetch `GET /api/avaliacoes/stats` + `GET /api/funcionarios?ativos=true` | Full-screen `Loader2` spinner + "Carregando dados..." text |
| Filter change | Re-fetch `GET /api/avaliacoes/stats` with new params | Content area shows spinner, filters remain interactive |
| API error | Display error message | Red border container with error text (`text-destructive`) |
| No data | Display empty state | "Nenhum dado disponível" + guidance text |

### 2. Filter Interactions

| Element | Behavior | Response |
|---------|----------|----------|
| Funcionário Select (dropdown) | On change → update `funcionario_id` param → re-fetch stats | New data loads, cards + chart update |
| Funcionário "Todos" option | Clears `funcionario_id` param → shows group data | Default view |
| Mês dropdown | On change → set `from`/`to` for full month → re-fetch | Same as period filter |
| "Personalizado" option | Enables manual date inputs | Date inputs become active |
| Date "De:" input | On change → update `from` → re-fetch (if `to` is set) | Content updates |
| Date "Até:" input | On change → update `to` → re-fetch (if `from` is set) | Content updates |

### 3. BarChart Interactions

| Element | Behavior | Response |
|---------|----------|----------|
| Bar hover | Tooltip appears | `ChartTooltip` with `{nome}: {media.toFixed(2)}` |
| Bar click | `onClick` → capture `funcionario_id` | **Deferred to Fase 4.4** — prepare `router.push` call but keep disabled or no-op |
| Empty chart | No data in `comparativo[]` | Show "Nenhum funcionário com avaliações no período" |

### 4. Interaction States Summary

| State | Condition | Visual |
|-------|-----------|--------|
| Loading | Fetch in progress | Centered `Loader2` spinner + text |
| Error | Fetch failure | Red bordered container + error message |
| Empty (no data) | Zero evaluations in period | Inline text, no cards/chart |
| Empty (no employees) | Zero active employees | "Nenhum funcionário ativo cadastrado" |
| Success | Data loaded | Full dashboard with cards + chart |
| Filtering active | Funcionário selected | Select shows chosen name, data filtered |
| All employees | "Todos" selected | Select shows "Todos", group data shown |

## Anti-Patterns to Avoid

1. **Do not recalculate averages client-side** — `media` is `GENERATED STORED`, use `AVG(media)` in SQL
2. **Do not forget active filter** — all queries MUST `JOIN funcionarios WHERE ativo = 1`
3. **Do not use variable Y-axis domain** — BarChart must have `domain={[0, 10]}` for scale consistency
4. **Do not sort comparative by average** — sort by `f.nome` to keep bar colors stable across renders
5. **Do not duplicate CATEGORIAS constant for the 4th time** — redefine locally in this module OR extract to shared constants file under `src/lib/` (preferred but discretionary)

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Employee with zero evaluations doesn't appear | Documented behavior — intentional per D4.3-14 |
| Many employees cause overflow in BarChart | Recharts handles overflow with scroll; consider `BarChart margin` or `aspect` ratio |
| Bar colors shift when data updates | Sort `comparativo` by `f.nome` (not by `media`) for stable color assignment |

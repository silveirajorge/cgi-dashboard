# Phase 4.3: Dashboard do Grupo — Research

**Researched:** 2026-06-07
**Domain:** Dashboard de KPIs de avaliação de desempenho — agregação SQL, gráficos comparativos, filtros
**Confidence:** HIGH

## Summary

O Dashboard do Grupo de Avaliação é uma página de visualização de KPIs coletivos a partir da tabela `avaliacoes` (já populada pela Fase 4.2). O padrão arquitetural replica exatamente o que foi estabelecido na Fase 3 (KPI Dashboard): página Server Component → Client Component filho → fetch client-side → componentes de cards, gráfico, filtros.

A diferença crítica é que este dashboard **não tem dados de pedidos como fonte**, mas sim avaliações de desempenho com categorias fixas (6 dimensões 1-10). A API de stats precisa agregar por `funcionario_id`, calcular médias por categoria, e produzir um array comparativo para o gráfico de barras. A coluna `media` já é `GENERATED STORED` — `AVG(media)` funciona diretamente no SQLite.

O gráfico Recharts BarChart com barras coloridas por funcionário segue o padrão `Bar` + `Cell` do `channel-chart.tsx`. A filtragem por período e funcionário segue o padrão do `period-filter.tsx` da Fase 3. O clique na barra deve preparar o `onClick` para navegação (Fase 4.4).

**Primary recommendation:** Clonar o padrão `kpi-dashboard-client.tsx` adaptando para o schema `avaliacoes`. Criar API route dedicada `GET /api/avaliacoes/stats`. Usar `Bar` + `Cell` do Recharts com cores `--chart-1` a `--chart-N` para o gráfico comparativo.

### Pattern 3: Fetch Pattern com useEffect + useState
**What:** Client component faz fetch de dados no useEffect com loading/error states + refetch ao mudar filtros.
**When to use:** Qualquer página que precise carregar dados de API.
**Source:** `src/app/(main)/dashboard/kpi/_components/kpi-dashboard-client.tsx`

```tsx
// Estrutura:
const [data, setData] = useState<StatsResponse | null>(null);
const [loading, setLoading] = useState(true);
const [error, setError] = useState<string | null>(null);

async function fetchStats(fromVal: string, toVal: string, funcionarioId?: string) {
  setLoading(true);
  setError(null);
  try {
    const params = new URLSearchParams({ from: fromVal, to: toVal });
    if (funcionarioId) params.set("funcionario_id", funcionarioId);
    const res = await fetch(`/api/avaliacoes/stats?${params}`);
    if (!res.ok) throw new Error("Erro ao carregar dados");
    setData(await res.json());
  } catch (err) {
    setError(err instanceof Error ? err.message : "Erro desconhecido");
  } finally {
    setLoading(false);
  }
}
```

### Anti-Patterns to Avoid
- **Recalcular médias no client:** A média já é `GENERATED STORED` no SQLite. Usar `AVG(media)` na query, nunca somar notas no frontend.
- **Esquecer filtro de ativos:** O JOIN com `funcionarios WHERE ativo = 1` é obrigatório em todas as queries da API stats.
- **BarChart sem scale fixa:** O eixo Y deve ter `domain={[0, 10]}` para que as barras representem corretamente a escala 1-10.
- **Duplicar constantes CATEGORIAS:** Já existem duas definições de `CATEGORIAS` no código (`avaliacao-form.tsx` e `avaliacao-detail-modal.tsx`). Extrair para um shared constant ou redefinir localmente.

## Common Pitfalls

### Pitfall 1: GENERATED STORED column em AVG()
**What goes wrong:** Assumir que `AVG(media)` não funciona porque `media` é GENERATED.
**Why it happens:** Confusão entre `STORED` e `VIRTUAL` no SQLite.
**How to avoid:** A coluna `media` é `GENERATED ... STORED` — existe fisicamente na tabela. `AVG(media)` funciona perfeitamente. Query verificada no schema: `media REAL GENERATED ALWAYS AS ((pontualidade + qualidade + produtividade + trabalho_equipa + iniciativa + comunicacao) / 6.0) STORED`.

### Pitfall 2: Funcionário com zero avaliações no período
**What goes wrong:** Funcionários ativos mas sem avaliações não aparecem no comparativo.
**Why it happens:** INNER JOIN com `avaliacoes` exclui funcionários sem registros.
**How to avoid:** É o comportamento desejado (D4.3-14: "apenas funcionários ativos entram nas métricas" — significa que apenas avaliações de funcionários ativos são consideradas, não que todos os ativos apareçam). Se um funcionário ativo não tem avaliações, ele não aparece. Documentar este comportamento.

### Pitfall 3: Cores inconsistentes no BarChart
**What goes wrong:** Funcionários trocam de cor quando a ordem do array muda (nova avaliação altera média, reposiciona).
**Why it happens:** Usar `data.indexOf(entry) % COLORS.length` num array ordenado por média.
**How to avoid:** Ordenar o comparativo por nome (`ORDER BY f.nome`) em vez de por média para que as cores sejam estáveis entre renders. Se a ordem for por média, aceitar que cores mudam.

### Pitfall 4: Category order mismatch
**What goes wrong:** API retorna categorias numa ordem, frontend espera noutra.
**Why it happens:** `SELECT AVG(...)` retorna na ordem da query, mas o frontend itera sobre `Object.entries`.
**How to avoid:** A API deve retornar `media_categorias` como um array ordenado `[{ key, label, value }]` em vez de objeto plano, ou garantir que o frontend use a mesma constante `CATEGORIAS` para iterar.

## Code Examples

### API Route: GET /api/avaliacoes/stats
```typescript
// Source: Adaptado de src/app/api/avaliacoes/route.ts + src/app/api/data/route.ts
import { type NextRequest, NextResponse } from "next/server";
import { getDb } from "@/lib/db";

export async function GET(request: NextRequest) {
  try {
    const { searchParams } = new URL(request.url);
    const from = searchParams.get("from");
    const to = searchParams.get("to");
    const funcionarioId = searchParams.get("funcionario_id");

    const db = getDb();
    const conditions: string[] = ["f.ativo = 1"];
    const params: (string | number)[] = [];

    if (from) { conditions.push("a.data_avaliacao >= ?"); params.push(from); }
    if (to) { conditions.push("a.data_avaliacao <= ?"); params.push(to); }
    if (funcionarioId) { conditions.push("a.funcionario_id = ?"); params.push(Number(funcionarioId)); }

    const where = `WHERE ${conditions.join(" AND ")}`;

    // Média geral
    const mediaGeralRow = db
      .prepare(`SELECT AVG(a.media) as media_geral FROM avaliacoes a JOIN funcionarios f ON f.id = a.funcionario_id ${where}`)
      .get(...params) as { media_geral: number | null };

    // Médias por categoria
    const mediaCategoriasRow = db
      .prepare(`
        SELECT
          AVG(a.pontualidade) as pontualidade,
          AVG(a.qualidade) as qualidade,
          AVG(a.produtividade) as produtividade,
          AVG(a.trabalho_equipa) as trabalho_equipa,
          AVG(a.iniciativa) as iniciativa,
          AVG(a.comunicacao) as comunicacao
        FROM avaliacoes a
        JOIN funcionarios f ON f.id = a.funcionario_id
        ${where}
      `).get(...params) as Record<string, number | null>;

    // Total de avaliações
    const totalRow = db
      .prepare(`SELECT COUNT(*) as total FROM avaliacoes a JOIN funcionarios f ON f.id = a.funcionario_id ${where}`)
      .get(...params) as { total: number };

    // Melhor funcionário (apenas quando não filtrado por funcionário)
    let melhorFuncionario: { id: number; nome: string; media: number } | null = null;
    if (!funcionarioId) {
      melhorFuncionario = db
        .prepare(`
          SELECT f.id, f.nome, AVG(a.media) as media
          FROM avaliacoes a
          JOIN funcionarios f ON f.id = a.funcionario_id
          ${where}
          GROUP BY f.id, f.nome
          ORDER BY media DESC
          LIMIT 1
        `).get(...params) as typeof melhorFuncionario | undefined ?? null;
    }

    // Comparativo (todos os funcionários com avaliações no período)
    const comparativo = db
      .prepare(`
        SELECT f.id as funcionario_id, f.nome as funcionario_nome, AVG(a.media) as media
        FROM avaliacoes a
        JOIN funcionarios f ON f.id = a.funcionario_id
        ${where}
        GROUP BY f.id, f.nome
        ORDER BY f.nome
      `).all(...params) as Array<{ funcionario_id: number; funcionario_nome: string; media: number }>;

    return NextResponse.json({
      media_geral: mediaGeralRow.media_geral ?? null,
      media_categorias: {
        pontualidade: mediaCategoriasRow.pontualidade ?? null,
        qualidade: mediaCategoriasRow.qualidade ?? null,
        produtividade: mediaCategoriasRow.produtividade ?? null,
        trabalho_equipa: mediaCategoriasRow.trabalho_equipa ?? null,
        iniciativa: mediaCategoriasRow.iniciativa ?? null,
        comunicacao: mediaCategoriasRow.comunicacao ?? null,
      },
      total_avaliacoes: totalRow.total,
      melhor_funcionario: melhorFuncionario,
      comparativo,
    });
  } catch (error) {
    const message = error instanceof Error ? error.message : "Erro interno";
    return NextResponse.json({ error: message }, { status: 500 });
  }
}
```

## Decisions Made

- **AvaliacaoFilters** combines both the funcionário dropdown and period filter into one component, avoiding a separate filter component for each
- **Todos value = ""** — the "Todos" option uses an empty string, making the client-side `if (funcionarioId)` check naturally skip the param
- **Bar onClick casting** — Recharts `BarRectangleItem` type doesn't include custom data fields; a type-safe cast `entry as { funcionario_id?: number }` is used, deferred to Fase 4.4
- **melhor_funcionario conditional** — only queries when no `funcionario_id` filter is active, since "melhor" is meaningless when filtering for one employee
- **Server Component title** — uses `sidebarItems.find(groupLabel)` pattern matching the Phase 3 KPI dashboard exactly

## Deviations from Plan

## Issues Encountered

- **Type error on Bar onClick:** Recharts `BarRectangleItem` type in v3.8.0 doesn't include custom data properties. Fixed by adding an explicit type cast `entry as { funcionario_id?: number }`. This does not change behavior — the runtime data contains the field, only the TypeScript type was wrong.

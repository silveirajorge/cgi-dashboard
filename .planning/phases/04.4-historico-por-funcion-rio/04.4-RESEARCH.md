# Phase 4.4: Histórico por Funcionário — Research

**Researched:** 2026-06-07
**Domain:** Dashboard — Modal de histórico de avaliações com gráfico LineChart + tabela de detalhes
**Confidence:** HIGH

## Summary

O Histórico por Funcionário é um modal (Dialog) aberto a partir do Dashboard de Avaliação (4.3), que mostra a evolução temporal das avaliações de um funcionário específico via LineChart (média geral) + tabela detalhada (data, notas por categoria, média, comentário). Apenas leitura, com filtro de período interno.

O códigobase já possui **todos os padrões necessários** para implementar esta fase: o `cil-modal.tsx` demonstra o padrão exato de modal com tabela e fetch condicional, o `daily-chart.tsx` demonstra o padrão de LineChart com gradiente e tooltip, e o `comparative-chart.tsx` já tem o `onBarClick` preparado.

**Descoberta crítica:** A API `GET /api/avaliacoes` já suporta filtragem por `funcionario_id` + `from`/`to` (não é necessário criar endpoint novo), mas **o SELECT atual não inclui as notas individuais por categoria** (apenas `data_avaliacao, media, comentario, funcionario_id, funcionario_nome`). Para a tabela de detalhes (D4.4-06) é necessário adicionar as 6 colunas de categoria ao SELECT.

**Primary recommendation:** Criar `historico-modal.tsx` seguindo o padrão do `cil-modal.tsx`, com LineChart seguindo o padrão do `daily-chart.tsx`, modificar o SELECT da API `/api/avaliacoes` para incluir categorias, e conectar no `avaliacao-dashboard-client.tsx`.

**What:** Modal que faz fetch dos dados apenas quando aberto, com loading/error/empty states e filtro interno.

### Anti-Patterns to Avoid
- **Criar endpoint novo:** Não é necessário — `GET /api/avaliacoes?funcionario_id=X` já existe. Basta modificar o SELECT.
- **Router.push para nova página:** D4.4-03 exige modal, não navegação.
- **Passar dados por props do server component:** O modal precisa de fetch próprio para suportar filtro de período interno.
- **Esquecer de incluir categorias no SELECT:** A resposta atual não tem `pontualidade, qualidade, produtividade, trabalho_equipa, iniciativa, comunicacao` — essenciais para a tabela detalhada.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Modal overlay | Dialog custom | shadcn/ui `Dialog` | Acessibilidade, focus trap, animação, teclado |
| Table | Table custom | shadcn/ui `Table` | Responsivo, acessível, padrão do projeto |
| Chart | Canvas/SVG custom | recharts `LineChart` | 266 code snippets, padrão do projeto |
| Chart wrapper | CSS custom | shadcn/ui `ChartContainer` | Tema integrado, tooltip, legend, responsivo |

**Key insight:** Todos os componentes necessários já existem no stack. O risco é querer otimizar ou personalizar o que já está pronto.

## Common Pitfalls

### Pitfall 1: API Response Missing Category Columns
**What goes wrong:** A tabela de detalhes (D4.4-06) precisa mostrar notas por categoria, mas a API `GET /api/avaliacoes` atual retorna apenas `id, data_avaliacao, media, comentario, funcionario_id, funcionario_nome`.
**Why it happens:** O endpoint foi criado na Fase 4.2 para a listagem geral de avaliações, que não precisava das categorias individuais.
**How to avoid:** Modificar o SELECT na linha 34 de `src/app/api/avaliacoes/route.ts` para incluir `a.pontualidade, a.qualidade, a.produtividade, a.trabalho_equipa, a.iniciativa, a.comunicacao`.
**Warning signs:** Tabela com colunas de categoria vazias ou dados insuficientes.

### Pitfall 2: onClick no Nome do Funcionário Inexistente
**What goes wrong:** D4.4-01 diz "clique no nome do funcionário abre o histórico" mas `KpiCards` não tem click handler no nome do melhor funcionário.
**Why it happens:** O `KpiCards` atual renderiza o nome num `<span>` simples sem onClick.
**How to avoid:** Adicionar prop `onFuncionarioClick?: (id: number) => void` ao `KpiCards` e um button/trigger no span do nome.
**Warning signs:** Ao clicar no nome do melhor funcionário, nada acontece.

### Pitfall 3: Modal Fora do Client Component Pai
**What goes wrong:** Tentar abrir o modal dentro de um server component ou sub-componente que não tem acesso ao estado do cliente pai.
**Why it happens:** O modal precisa de estado booleano (`isOpen`) que vive no `avaliacao-dashboard-client.tsx`.
**How to avoid:** Manter todo o estado do modal no `avaliacao-dashboard-client.tsx`, renderizar o modal condicionalmente no mesmo nível que os outros componentes.
**Warning signs:** Modal não abre ou abre no lugar errado.

### Pitfall 4: Re-render do Modal Força Refetch Indesejado
**What goes wrong:** Mudar o filtro de período no dashboard pai (fora do modal) força re-render do modal e refetch desnecessário.
**Why it happens:** O modal recebe `from`/`to` como props do pai que mudam com o filtro global.
**How to avoid:** O modal deve controlar seu próprio `from`/`to` internamente (D4.4-08 — filtro por período dentro do modal). Não herdar do pai.
**Warning signs:** Modal fecha sozinho ou refaz fetch quando altera filtro do dashboard.

### HistoricoModal — Estrutura Completa

```typescript
"use client";

import { useCallback, useEffect, useState } from "react";
import { format, parseISO } from "date-fns";
import { Loader2 } from "lucide-react";
import { Area, CartesianGrid, Line, LineChart, XAxis, YAxis } from "recharts";

import {
  type ChartConfig,
  ChartContainer,
  ChartTooltip,
  ChartTooltipContent,
} from "@/components/ui/chart";
import { Dialog, DialogContent, DialogHeader, DialogTitle } from "@/components/ui/dialog";
import { Table, TableBody, TableCell, TableHead, TableHeader, TableRow } from "@/components/ui/table";

// Source: Padrão cil-modal.tsx + daily-chart.tsx combinados

const CATEGORIAS = [
  { key: "pontualidade", label: "Pontualidade" },
  { key: "qualidade", label: "Qualidade" },
  { key: "produtividade", label: "Produtividade" },
  { key: "trabalho_equipa", label: "Trabalho em Equipa" },
  { key: "iniciativa", label: "Iniciativa" },
  { key: "comunicacao", label: "Comunicação" },
] as const;

interface AvaliacaoRecord {
  id: number;
  data_avaliacao: string;
  pontualidade: number;
  qualidade: number;
  produtividade: number;
  trabalho_equipa: number;
  iniciativa: number;
  comunicacao: number;
  media: number;
  comentario: string | null;
  funcionario_id: number;
  funcionario_nome: string;
}

interface HistoricoModalProps {
  funcionarioId: number;
  funcionarioNome: string;
  onClose: () => void;
}

export function HistoricoModal({ funcionarioId, funcionarioNome, onClose }: HistoricoModalProps) {
  const [records, setRecords] = useState<AvaliacaoRecord[]>([]);
  const [loading, setLoading] = useState(false);
  const [from, setFrom] = useState("");
  const [to, setTo] = useState("");

  const fetchHistorico = useCallback(async (fromVal: string, toVal: string) => {
    setLoading(true);
    try {
      const params = new URLSearchParams({ funcionario_id: String(funcionarioId) });
      if (fromVal) params.set("from", fromVal);
      if (toVal) params.set("to", toVal);
      const res = await fetch(`/api/avaliacoes?${params}`);
      if (!res.ok) throw new Error("Erro ao carregar histórico");
      const json: AvaliacaoRecord[] = await res.json();
      setRecords(json);
    } catch {
      setRecords([]);
    } finally {
      setLoading(false);
    }
  }, [funcionarioId]);

  useEffect(() => {
    void fetchHistorico(from, to);
  }, [fetchHistorico, from, to]);

  const chartConfig = {
    media: { label: "Média Geral", color: "var(--chart-1)" },
  } satisfies ChartConfig;

  // Dados ordenados por data ascendente para o gráfico
  const chartData = [...records].sort(
    (a, b) => a.data_avaliacao.localeCompare(b.data_avaliacao),
  );

  return (
    <Dialog open={funcionarioId !== null} onOpenChange={(open) => { if (!open) onClose(); }}>
      <DialogContent className="sm:max-w-6xl">
        <DialogHeader>
          <DialogTitle>{funcionarioNome} &mdash; {records.length} avaliações</DialogTitle>
        </DialogHeader>

        {/* Filtro de período interno (date range) */}
        <div className="flex items-center gap-2">
          <span className="text-muted-foreground text-sm">De:</span>
          <input type="date" value={from}
            onChange={(e) => setFrom(e.target.value)}
            className="h-8 rounded-lg border border-input bg-transparent px-2.5 text-sm" />
          <span className="text-muted-foreground text-sm">Até:</span>
          <input type="date" value={to}
            onChange={(e) => setTo(e.target.value)}
            className="h-8 rounded-lg border border-input bg-transparent px-2.5 text-sm" />
        </div>

        {loading ? (
          <div className="flex items-center justify-center py-12">
            <Loader2 className="size-6 animate-spin text-muted-foreground" />
          </div>
        ) : (
          <>
            {/* LineChart — Evolução da Média Geral */}
            <ChartContainer config={chartConfig} className="h-80 w-full">
              <LineChart data={chartData} margin={{ top: 5, right: 10, left: 10, bottom: 0 }}>
                <defs>
                  <linearGradient id="fillMedia" x1="0" y1="0" x2="0" y2="1">
                    <stop offset="5%" stopColor="var(--color-media)" stopOpacity={0.36} />
                    <stop offset="95%" stopColor="var(--color-media)" stopOpacity={0.04} />
                  </linearGradient>
                </defs>
                <CartesianGrid vertical={false} strokeOpacity={0.5} />
                <XAxis
                  dataKey="data_avaliacao"
                  tickLine={false} axisLine={false} tickMargin={8} fontSize={12}
                  tickFormatter={(d: string) => format(parseISO(d), "dd/MM")}
                />
                <YAxis domain={[0, 10]} tickLine={false} axisLine={false} tickMargin={8} fontSize={12} />
                <ChartTooltip
                  content={<ChartTooltipContent
                    labelFormatter={(_l, p) => {
                      const d = (p as Array<{ payload: AvaliacaoRecord }>)?.[0]?.payload?.data_avaliacao;
                      return d ? format(parseISO(d), "dd 'de' MMMM 'de' yyyy") : "";
                    }}
                  />}
                  cursor={{ stroke: "var(--border)", strokeDasharray: "4 4" }}
                />
                <Area dataKey="media" type="natural" fill="url(#fillMedia)" stroke="var(--color-media)" strokeWidth={1.25} dot={{ r: 3 }} />
                <Line dataKey="media" type="natural" stroke="var(--color-media)" strokeWidth={1.5} dot={{ r: 3 }} />
              </LineChart>
            </ChartContainer>

            {/* Tabela de Detalhes */}
            <div className="max-h-[40vh] overflow-y-auto">
              <Table>
                <TableHeader>
                  <TableRow>
                    <TableHead>Data</TableHead>
                    {CATEGORIAS.map((cat) => (
                      <TableHead key={cat.key} className="text-center">{cat.label}</TableHead>
                    ))}
                    <TableHead className="text-center">Média</TableHead>
                    <TableHead>Comentário</TableHead>
                  </TableRow>
                </TableHeader>
                <TableBody>
                  {records.length === 0 ? (
                    <TableRow>
                      <TableCell colSpan={9} className="text-center text-muted-foreground">
                        Nenhuma avaliação encontrada
                      </TableCell>
                    </TableRow>
                  ) : (
                    records.map((r) => (
                      <TableRow key={r.id}>
                        <TableCell>{format(parseISO(r.data_avaliacao), "dd/MM/yyyy")}</TableCell>
                        {CATEGORIAS.map((cat) => (
                          <TableCell key={cat.key} className="text-center">{r[cat.key]}</TableCell>
                        ))}
                        <TableCell className="text-center font-medium">{r.media.toFixed(2)}</TableCell>
                        <TableCell className="max-w-48 truncate">{r.comentario ?? "—"}</TableCell>
                      </TableRow>
                    ))
                  )}
                </TableBody>
              </Table>
            </div>
          </>
        )}
      </DialogContent>
    </Dialog>
  );
}
```
